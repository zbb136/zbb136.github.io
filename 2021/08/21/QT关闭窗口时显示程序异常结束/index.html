# QT 关闭窗口时显示程序 “异常结束” 解决办法

# 1. 问题概述

使用 QT 运行程序时，程序正常运行，但在关闭窗口后，程序输出窗口发现程序并没有立即关闭，而是等待一会儿后打印输出 “程序异常结束 “，若下图所示：

有人给出了一种暴力解决方案：修改 main.cpp 函数中的这两行代码

//	原代码
//    MainWindow w;
//    w.show();

//	修改后
    MainWindow* w=new MainWindow;
    w->show();

emm… 确实可以让它不再报错了，可是如果你在析构函数里放个打印输出函数的话，你会发现你关闭窗口后它并不会给你打印，也就是说你关闭窗口时它并不执行析构函数！一些 new 出来的变量内存并不能很好地释放掉，在 QT 中运行还好，但如果你发布成软件的话在进行关闭软件操作时就会出现问题，可以说这并没有解决真正的问题。下面介绍产生这种报错的原因的解决方法。

注：本文分析的是关闭窗口后报” 程序异常结束 “错误，而不是程序刚开始运行或运行过程中报错，后者产生的原因有很多，建议点击 QT 中的调试 -> 开始调试 ->start debuging for…，运行时会直接给你找出出错的语句

# 2. 问题产生原因与解决方法

# ①对象被多次析构

存在这种问题时，如果关闭窗口，程序输出窗口会等 2~3s 就会报” 程序异常结束 “，那么大概率是对象被多次析构了，一下情况会导致对象被多次析构：

/* 1.new一个新对象时给它配置了父对象，而在别处或析构函数中delete了它的对象指针
	实际上，给某对象配置父对象后，父对象销毁时该对象指针会被自动销毁，无需去主动销毁它
	但如果new的是一个子线程，则必须主动销毁它，销毁方式见②*/
//如：
QCPItemRect* selectRect=new QCPItemRect(this);
delete selectRect;

/* 2.试图去delete一个智能指针
	智能指针不需要去主动销毁*/
//如：
shared_ptr<InterfaceForPython> p_obj_python=make_shared<InterfaceForPython>();
delete p_obj_python;

/* 3.同一个对象指针被delete了多次*/
//如：
QCPItemRect* selectRect=new QCPItemRect(this);
delete selectRect;
delete selectRect;	//最后这次delete一定是在某个析构函数中，不然运行中就会报错

解决办法：当然是找出在哪里用了不该用的 delete 然后删掉咯

# ②存在 while 死循环或耗时任务无法在短时间内运行完毕，或多线程对象指针没有主动销毁

存在这种问题时，如果关闭窗口，程序输出窗口会等待很长时间，然后报” 程序异常结束 “。这个就不用多说了，尤其多线程中可能就需要 while 死循环或存在耗时任务来不断执行某些代码。

解决办法如下：

/*若是存在while死循环或耗时任务无法在短时间内运行完毕，需要在循环任务中添加跳出标志位，在该函数所在对象的父对象的析构函数中修改标志位的值，达到强制跳出循环的目的*/
class Myclass: QObject
{
    Myclass(QObject *parent = nullptr);
    void noStopRun(){while(runflag){...}	};	//runflag是跳出标志位,noStopRun()函数中while(runflag){...}是一个死循环或耗时任务
    bool runflag=true;
}

MainWindow::MainWindow(QWidget *parent)
    : QMainWindow(parent)
    , ui(new Ui::MainWindow)
{
        myObject.noStopRun();	//myObject为在MainWindow的类声明中定义的成员变量：Myclass myObject;
}
MainWindow::~MainWindow()
{
    myObject.runflag=false;	//使子函数退出死循环
}

/*若是多线程对象指针没有主动销毁，按此步骤销毁*/
MainWindow::~MainWindow()	
{
    //若存在while死循环或耗时任务无法在短时间内运行完毕，按上面方式添加标志位，并在此赋false；若不存在则不需要这行代码
    //thread_manager->runflag=false;	
    
    thread_manager->quit();	//thread_manager是一个子线程
    thread_manager->wait();  //必须等待线程结束
    thread_manager->deleteLater();
    delete ui;
    delete thread_manager;
}
